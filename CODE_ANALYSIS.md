## 5. Security Features Implemented

| Feature | Implementation | Location |
|---|---|---|
| **JWT Authentication** | `jsonwebtoken`, 7-day expiry, signed with `JWT_SECRET` | `User.generateToken()` |
| **HTTP-Only Cookies** | `httpOnly: true`, prevents XSS token theft | `cookieOptions.js` |
| **Password Hashing** | `bcryptjs`, 10 salt rounds, pre-save hook | `User.js` pre-save |
| **Password Never Returned** | `select: false` on password field | `User.js` schema |
| **Role-Based Access Control** | `authorize()` middleware checks `req.user.role` | `role.middleware.js` |
| **CORS Whitelist** | Explicit `allowedOrigins` array | `app.js` |
| **Double Booking Prevention** | Application check + MongoDB unique compound index | `appointment.controller.js` + `Appointment.js` |
| **Patient Self-Conflict Check** | Prevents same patient booking same slot twice with different doctors | `appointment.controller.js` |
| **Past Appointment Protection** | Cannot book or cancel appointments in the past | `appointment.controller.js`, `cancelAppointment()` |
| **Authorization Check on Cancel** | Verifies `appointment.patient === req.user._id` | `cancelAppointment()` |
| **Email Validation** | Regex validation before user lookup | `auth.controller.js` |
| **Cookie Clearing on Logout** | `res.clearCookie()` with matching options | `auth.controller.js` |
| **Credentials in CORS** | `credentials: true` for cookie-based auth | `app.js` |

## 6. Performance Optimizations

| Optimization | Details | Location |
|---|---|---|
| **Compound Unique Index** | `{doctor, appointmentDate, startTime}` index for O(log n) conflict detection | `Appointment.js` |
| **Selective Field Projection** | `populate("user", "name email")` avoids over-fetching | All controllers |
| **Password Excluded by Default** | `select: false` prevents password in every query | `User.js` |
| **Normalized Dates** | `setUTCHours(0,0,0,0)` prevents timezone-based duplicate bookings | `appointment.controller.js` |
| **Graceful AI Fallback** | AI failures don't block critical paths (booking still completes) | `ai.service.js` |
| **Non-blocking Email** | Email errors are caught and logged without blocking main response | `appointment.controller.js` |
| **Non-blocking Calendar** | Calendar errors don't block booking success | `appointment.controller.js` |
| **Patient grouping in Cron** | Groups medications by patient email to send one email per patient | `reminder.job.js` |
| **Sorted Queries** | Appointments sorted by date (`-1` for patients, `1` for doctors) | `appointment.controller.js`, `doctor.controller.js` |
| **React Query Caching** | TanStack Query caches API responses on frontend | Frontend services |
| **Vite Build Tool** | Fast HMR in development, optimized production bundles | `vite.config.js` |

## 9. Code Quality Notes

| Observation | Details |
|---|---|
| **Consistent error handling** | All controllers return standardized `{success, message}` JSON |
| **Async/await** | All async operations use modern async/await (no callback hell) |
| **Graceful degradation** | AI and email failures are caught and do not block primary operations |
| **Separation of concerns** | Services layer cleanly separates business logic from HTTP handling |
| **Dynamic imports** | Some controllers use `await import(...)` for circular dependency avoidance |
| **IST timezone awareness** | All date comparisons explicitly convert to IST for correct behavior in India |
| **Template functions** | 15 email template functions with a shared `baseTemplate` pattern |
| **No TypeScript** | JavaScript-only; adding TypeScript would improve type safety |
| **No test suite** | No automated tests currently exist — highest priority improvement |
